Как работать в GitHub

В этом курсе вы научитесь, как использовать GitHub, одну из самых популярных и мощных платформ для совместной разработки проектов. Вы узнаете, как создавать репозитории, вносить изменения, синхронизировать код, работать с ветками и слияниями, а также как взаимодействовать с другими разработчиками.



Курс состоит из нескольких уроков, в каждом из которых вы будете изучать новые аспекты работы с GitHub. В конце каждого урока вы найдете задания, которые помогут вам практиковать использование GitHub и совместную разработку проектов. Также вы сможете получить доступ к репозиториям и материалам, которые использовались в уроке. Для прохождения курса вам понадобятся базовые знания работы с командной строкой и системами контроля версий. Если вы новичок в этих областях, мы рекомендуем вам сначала пройти наши вводные курсы по командной строке и Git, которые вы можете найти на нашем сайте.


Урок 1: Как создать и настроить репозиторий на GitHub

В этом уроке вы узнаете, что такое репозиторий и как его создать на GitHub. Репозиторий - это место, где хранятся и управляются файлы вашего проекта. Вы также научитесь, как настроить репозиторий, добавить описание, лицензию, README файл и игнорировать ненужные файлы.



После просмотра видео вы сможете выполнить следующие задания:

  • Создайте свой первый репозиторий на GitHub с именем hello-world.
  • Добавьте краткое описание вашего репозитория в поле Description.
  • Создайте файл README.md в вашем репозитории и напишите в нем "Hello, world!"
  • Создайте файл .gitignore в вашем репозитории и добавьте в него "*.txt", чтобы игнорировать все текстовые файлы.


Урок 2: Как вносить изменения в репозиторий на GitHub

В этом уроке вы узнаете, как вносить изменения в файлы вашего репозитория на GitHub. Изменения - это действия, которые вы выполняете над файлами, такие как создание, редактирование, удаление и перемещение. Вы также научитесь, как фиксировать изменения, писать осмысленные сообщения коммитов и просматривать историю коммитов.



После просмотра видео вы сможете выполнить следующие задания:

  • Отредактируйте файл README.md в вашем репозитории и добавьте в него "This is my first repository on GitHub."
  • Создайте новый файл index.html в вашем репозитории и напишите в нем базовую разметку HTML.
  • Удалите файл .gitignore из вашего репозитория.
  • Переместите файл index.html в папку src, которую вы создадите в вашем репозитории.
  • Зафиксируйте все изменения в вашем репозитории с сообщением "Add index.html and edit README.md"
  • Просмотрите историю коммитов в вашем репозитории и убедитесь, что все изменения отражены.


Урок 3: Как синхронизировать код между локальным и удаленным репозиторием на GitHub

В этом уроке вы узнаете, как синхронизировать код между локальным и удаленным репозиторием на GitHub. Локальный репозиторий - это копия вашего проекта, которая хранится на вашем компьютере. Удаленный репозиторий - это копия вашего проекта, которая хранится на GitHub. Вы также научитесь, как клонировать репозиторий, отправлять и получать изменения, и решать конфликты.



После просмотра видео вы сможете выполнить следующие задания:

  • Клонируйте ваш репозиторий hello-world на свой локальный компьютер с помощью команды `git clone` и URL вашего репозитория, который вы можете скопировать с GitHub.
  • Откройте папку с вашим локальным репозиторием в любом текстовом редакторе и отредактируйте файл README.md, добавив в него "This is my first repository on GitHub."
  • Отправьте изменения в ваш локальный репозиторий с помощью команды `git commit -m "Edit README.md"`.
  • Отправьте изменения в ваш удаленный репозиторий на GitHub с помощью команды `git push`.
  • Откройте ваш репозиторий на GitHub и убедитесь, что изменения отображаются.
  • Отредактируйте файл README.md в вашем репозитории на GitHub, добавив в него "This is a change made on GitHub."
  • Получите изменения из вашего удаленного репозитория на GitHub в ваш локальный репозиторий с помощью команды `git pull`.
  • Откройте файл README.md в вашем локальном репозитории и убедитесь, что изменения отображаются.
  • Создайте конфликт в файле README.md, добавив в него разные строки в вашем локальном и удаленном репозитории.
  • Попробуйте отправить изменения из вашего локального репозитория на GitHub с помощью команды `git push` и увидите, что произойдет.
  • Попробуйте получить изменения из вашего удаленного репозитория на GitHub в ваш локальный репозиторий с помощью команды `git pull` и увидите, как решить конфликт.


Урок 4: Как работать с ветками на GitHub

В этом уроке вы узнаете, что такое ветки и как с ними работать на GitHub. Ветки - это параллельные версии вашего проекта, которые позволяют вам вносить изменения без влияния на основную ветвь master. Вы также научитесь, как создавать, переключаться, сливать и удалять ветки.



После просмотра видео вы сможете выполнить следующие задания:

  • Создайте новую ветку в вашем локальном репозитории с именем feature и переключитесь на нее с помощью команды `git checkout -b feature`.
  • Отредактируйте файл index.html в вашей ветке feature, добавив в него тег `This is a new feature.`.
  • Отправьте изменения в ваш локальный репозиторий с помощью команды `git commit -m "Add new feature"`.
  • Отправьте изменения в ваш удаленный репозиторий на GitHub с помощью команды `git push -u origin feature`.
  • Откройте ваш репозиторий на GitHub и убедитесь, что появилась новая ветка feature.
  • Создайте запрос на слияние (pull request) из ветки feature в ветку master на GitHub и добавьте описание вашего изменения.
  • Просмотрите и одобрите запрос на слияние на GitHub и убедитесь, что изменения из ветки feature добавились в ветку master.
  • Удалите ветку feature из вашего локального и удаленного репозитория с помощью команд `git branch -d feature` и `git push origin --delete feature`.
